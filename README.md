# mine_safety_agent
构建了一个基于多 Agent 协作的【智慧矿山电气安全合规审计系统】。该项目解决了长距离大负荷工况下（如 1200 米 500kW 负荷），人工核算压降与安全规程匹配极易出错的痛点。

🏭 SmartMine Safety Agent (智慧矿山安全合规审计系统)
基于多智能体协同与长链推理的工业安全闭环解决方案

本项目是一个针对智慧矿山领域设计的 AI Agent 原型。它能够深度结合电气专业参数（如长距离压降校核）与安全生产法规，将非结构化的现场隐患描述转化为标准化的、合规的审计整改报告。

🎯 解决的核心痛点
在传统的矿山安全管理中，存在以下三个“信息鸿沟”：

法规匹配难：安全规程（如《煤矿安全规程》）条文繁杂，一线人员难以在工况异常时迅速定位准确条款。

计算校核烦：长距离大负荷（如 1200米、500kW）的电气压降计算往往滞后于现场检查，导致隐患发现不及时。

经验依赖重：安全审计高度依赖资深专家的个人经验，缺乏标准化的逻辑推理过程。

⚙️ 核心逻辑流 (Core Workflow)
系统采用了 多智能体 (Multi-Agent) 协同架构，通过思维链 (CoT) 实现从“知”到“行”的转化：

🕵️ 解析 Agent (Parser)：从巡检日志中自动识别受控对象（如带式输送机）及关键电气工况参数。

📚 合规 Agent (Compliance)：利用 RAG（检索增强生成）技术，在海量安全规程库中精准锚定强制性合规条款。

🧮 计算 Agent (Tool Use)：自主调用后端 Python 计算脚本，针对电压降、负载率等硬指标进行工程校验。

🧠 决策 Agent (Decision Maker)：综合现场数据与计算结果，进行长链逻辑推理，输出带整改指令的标准化报告。

🚀 运行示例
用户输入：

"报告，二号井下主巷道 1200米 处的 500kW 皮带机防滑保护灯不亮了，而且电压掉得厉害。"

Agent 审计报告输出：

合规依据：引用《煤矿安全规程》第 373 条（关于防滑保护的规定）。

技术校核：计算显示末端压降超标（违规），判定隐患等级为“高”。

整改指令：建议立即停机排查传感器接线并校核电缆截面积。

📂 项目结构
mine_safety_agent.py: 核心多 Agent 协同逻辑。

main.py: 基于 FastAPI 的设备状态模拟后端。

index.html: 实时监测大屏前端（通过 WebSocket 接收模拟数据）。

requirements.txt: 项目环境依赖。

🛠️ 快速开始
Bash
# 安装依赖
pip install -r requirements.txt

# 运行审计 Agent 测试
python mine_safety_agent.py

# 启动模拟大屏后端
uvicorn main:app --reload
